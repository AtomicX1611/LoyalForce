## 1. ROLE & ARCHITECTURAL OBJECTIVE
You are an expert Full-Stack Software Engineer, Data Engineer, and AI Systems Architect. Your objective is to build a production-ready, scalable B2B SaaS platform for airline marketing managers to predict customer churn, identify Customer Lifetime Value (CLV) segments, and deploy targeted retention campaigns.

### Core Architectural Mandates:
* STRICT DECOUPLING: The frontend (ReactJS) and backend (Python/FastAPI) must remain completely separate, communicating exclusively via asynchronous JSON payloads over secure REST APIs.
* ZERO HARDCODING: Do not hardcode database connection strings, application secret keys, third-party authentication tokens, routing parameters, or external host URLs. Everything must be loaded dynamically from environment variables (.env).
* ISOLATED MULTI-TENANCY: The platform must enforce strict logical data isolation. You must implement a FastAPI Dependency (`get_current_tenant`) that extracts the `tenant_id` from the JWT. This dependency must be injected into all protected routes, guaranteeing that cross-tenant data leakage is structurally impossible.

## 2. OPERATIONALIZING THE MACHINE LEARNING PIPELINE
The machine learning logic currently resides in a Jupyter Notebook. Extract this logic using a Hybrid Batch + On-Demand Inference Architecture:

1. Model Artifact Export: The Jupyter Notebook must serialize the final trained pipeline into a standalone file using joblib (e.g., `loyalty_model.joblib`).
2. Batch Pre-computation (Dashboard Layer): Bulk metrics and persona clustering for the baseline historical dataset (16,700 customers) are pre-calculated offline. The resulting data is seeded directly into MongoDB for instant data grid load times.
3. On-Demand Inference Engine (Live Layer): 
   * CRITICAL TRADE-OFF AVOIDANCE: Scikit-learn predictions are synchronous and CPU-bound. To prevent blocking FastAPI's asynchronous event loop, the live inference route (`/api/predict/what-if`) MUST execute the `model.predict_proba()` call inside a background thread using `fastapi.concurrency.run_in_threadpool` or `asyncio.to_thread()`.
   * The FastAPI server must load the `loyalty_model.joblib` file into global memory exactly ONCE on system startup using an application lifespan event handler (`@asynccontextmanager`).

## 5. JSON REST API ROUTE SPECIFICATIONS

### Authentication Layer
* `POST /api/auth/mock-login`
  * Ingests JSON: `{ "email": "manager@nlair.com", "airline_id": "northern_lights_air" }`
  * Returns JSON: `{ "token": "JWT_STRING", "user": { "email": "manager@nlair.com", "tenant_id": "northern_lights_air" } }`

### Aggregation & Data Layer (JWT Guard & Tenant Dependency Required)
* `GET /api/dashboard/metrics`
  * Returns JSON: Computed metrics aggregate summary object localized to the tenant.
* `GET /api/customers`
  * Query Parameters: `page` (int), `limit` (int), `segment` (string, optional)
  * Returns JSON: Paginated array. 
  * Implementation Detail: To ensure scalable performance, avoid relying purely on unindexed `$skip` operators. Ensure the query utilizes the compound indexes.
* `PATCH /api/customers/{member_id}/campaign`
  * Ingests JSON: `{ "action_title": "Send Lounge Pass", "incentive": "10k Points" }`
  * Returns JSON: Mutates customer tracking state to "Campaign Active" and pushes an audit entry to `campaigns`.

### Dynamic Machine Learning Layer (JWT Guard Required)
* `POST /api/predict/what-if`
  * Ingests JSON: `{ "flight_freq_delta": float, "points_redeemed_ratio": float, "months_inactive": int }`
  * Returns JSON: `{ "recalculated_churn_risk": float }`
  * Implementation Detail: Pass incoming request fields into the feature structure expected by the in-memory `joblib` model. MUST execute in a threadpool to prevent event loop blocking.
